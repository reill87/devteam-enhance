# 개발팀 강화하기 PRD (가칭)

> 검 강화 게임 메커니즘 + 개발팀 직군 풍자 컨셉의 웹 게임
> 작성일: 2026-05-04
> 작성자: Lucas
> 성격: 개인 취미 프로젝트 (게임 완성 자체가 목표)

---

## 1. 프로젝트 정의

### 무엇을 만드는가
"검 강화하기" 류 게임의 메커니즘을 차용해서, **개발팀 캐릭터(기획자/디자이너/개발자)를 강화하는** 웹 게임.

### 왜 만드는가
- **게임을 처음부터 끝까지 완성해보는 경험** (1차 목표)
- **게임 엔진(Phaser) 학습** (2차 목표)
- 개발팀 직군 밈을 활용한 풍자적 재미 (있으면 좋음)

### 안 하는 것 (Non-Goals)
- 유저 획득, 트래픽, 수익화 — 신경 안 씀
- 교육 콘텐츠, 전문 코스 — 강화 메커니즘과 안 맞음, 제외
- 멀티플레이, 실시간 대결 — 범위 밖
- 모바일 앱 — 웹만 (모바일 브라우저는 지원)
- 백엔드 — MVP는 로컬(localStorage)만, 리더보드는 v2

## 2. 핵심 게임 루프

### 기본 루프
```
1. 캐릭터 선택 (기획자/디자이너/개발자 중 하나)
2. 재화 획득 (시간 경과 or 미니게임 or 광고 시청 패러디)
3. 강화 시도
   ├─ 성공 → 레벨업, 능력치 상승, 새 칭호/이펙트
   └─ 실패 → 강화 단계 하락 or 캐릭터 소멸 (난이도에 따라)
4. 강화 단계 도달 시 새 콘텐츠 해금 (직급 승진, 새 캐릭터, 도전 모드)
5. 다시 2번으로
```

### 강화 단계 예시 (개발자 트랙)

| 단계 | 직급 | 강화 성공률 | 실패 시 |
|------|------|------------|---------|
| 0 | 신입 개발자 | 100% | - |
| 1 | 주니어 개발자 | 95% | 단계 유지 |
| 2 | 중급 개발자 | 80% | 단계 유지 |
| 3 | 시니어 개발자 | 60% | 단계 -1 |
| 4 | 테크리드 | 40% | 단계 -1 |
| 5 | 스태프 엔지니어 | 25% | 단계 -2 |
| 6 | 프린시플 | 15% | 캐릭터 소멸 |
| 7 | 디스팅귀시드 | 8% | 캐릭터 소멸 |
| 8 | CTO | 3% | 캐릭터 소멸 |
| 9 | 전설의 풀스택 | 1% | 캐릭터 소멸 |
| 10 | 갓 개발자 | 0.5% | 캐릭터 소멸 |

(기획자/디자이너 트랙도 동일 구조, 직급명만 다름)

### 2.1 강화 메시지 (직군 밈 활용 - 핵심 차별화)

성공/실패 메시지를 직군별로 다르게. 게임의 재미 포인트.

**개발자 강화 실패 예시**
- "프로덕션에서 NullPointerException이 발생했습니다."
- "금요일 오후 6시에 배포한 코드가 롤백됐습니다."
- "코드 리뷰에서 50개의 코멘트를 받았습니다."
- "Stack Overflow가 다운됐습니다."

**기획자 강화 실패 예시**
- "요구사항이 또 바뀌었습니다."
- "임원이 데모를 보고 한 마디 했습니다: '이거 왜 이래요?'"
- "'간단한 거잖아요'라는 말을 들었습니다."

**디자이너 강화 실패 예시**
- "시안이 4번째 반려됐습니다."
- "'좀 더 임팩트 있게' 라는 피드백을 받았습니다."
- "개발자가 '이거 구현 안 돼요' 라고 했습니다."

**성공 메시지**
- "사이드 프로젝트가 GitHub 트렌딩에 올랐습니다!"
- "테크 블로그가 입소문이 났습니다."
- "스카웃 메일이 쇄도합니다."

## 3. 화면 구성 (MVP)

### 3.1 메인 화면
- 중앙: 현재 캐릭터 (강화 단계 표시)
- 상단: 보유 재화, 강화 횟수
- 하단: [강화하기] 버튼, [재화 획득] 버튼
- 좌측: 캐릭터 변경
- 우측: 도감/통계

### 3.2 강화 연출
- 강화 버튼 클릭 → 텐션 빌드업 애니메이션 (2~3초)
- 결과 발표: 성공 (이펙트 + 효과음) / 실패 (이펙트 + 효과음)
- 결과 메시지 표시
- 다시 메인으로 복귀

### 3.3 도감/통계
- 지금까지 도달한 최고 단계 (캐릭터별)
- 누적 강화 시도 / 성공 / 실패 횟수
- 누적 캐릭터 소멸 횟수
- 발견한 메시지 컬렉션

### 3.4 (선택) 결과 공유
- 캐릭터 강화 결과 카드 이미지 생성 → 다운로드
- "나는 시니어에서 떨어져서 신입이 됐다" 같은 SNS 공유용

## 4. 기술 스택

### 게임 엔진: **Phaser 3** 추천
| 비교 | Phaser 3 | PixiJS |
|------|----------|--------|
| 타입 | 게임 프레임워크 (씬, 인풋, 사운드 포함) | 렌더러 (그리기만) |
| 학습 곡선 | 중간 - 게임 개념 학습 가능 | 낮음 - 그리기만 하면 됨 |
| 이 게임 적합도 | 좋음 - 씬 전환, 사운드, UI 다 제공 | 직접 다 만들어야 함 |
| 자료 | 풍부 (게임 튜토리얼 많음) | 그래픽 위주 자료 |

→ **Phaser 3 채택**. "게임 엔진 학습" 목표에도 더 부합.

### 전체 스택
- **게임 엔진**: Phaser 3
- **번들러**: Vite
- **언어**: TypeScript
- **스타일/UI 외부**: 최소한의 HTML/CSS (랜딩, 메뉴 등)
- **저장소**: localStorage (MVP)
- **에셋**:
  - 캐릭터 일러스트: 직접 그리거나 SVG 도형 조합으로 시작 → 나중에 AI 이미지 생성 활용
  - 효과음: [freesound.org](http://freesound.org) 또는 직접 생성
  - 폰트: 무료 한글 폰트 (Pretendard 등)
- **배포**: Vercel 또는 Cloudflare Pages (정적 호스팅)

## 7. 게임 디자인 결정사항

### 7.1 강화 실패 페널티
- **선택**: 단계별 차등 페널티 (낮은 단계는 유지, 높은 단계는 -1, 최상위는 소멸)
- **이유**: 너무 자주 소멸하면 짜증, 너무 안 소멸하면 긴장감 없음
- **최종 단계 도달은 매우 어렵게**: 검 강화의 핵심 재미

### 7.2 재화 획득 방법
- **시간 경과 (자동 회복)**: 기본 - 매 X초마다 1씩
- **광고 시청 패러디 미니게임**: "광고 보기" 버튼 누르면 진짜 광고 같은 미니게임 (직군 밈 활용)
- **첫 캐릭터 소멸 보너스**: 신규 시작 시 재화 부여
- **MVP**: 시간 경과 + 클릭 보너스만, 미니게임은 v2

### 7.3 영구 진행감 (Meta Progression)
- 캐릭터가 소멸해도 **누적 통계, 도감, 최고 기록**은 남음
- "지금까지 N번 폭사했지만 결국 갓 개발자에 도달했다" 같은 서사
- 일정 단계 이상 도달 시 새 캐릭터 외형/색상 해금 (선택)

### 7.4 사운드
- 강화 빌드업: 긴장감 있는 톤
- 성공: 띠링~ 한국 모바일 게임 느낌
- 실패: 절망적인 효과음
- BGM: 잔잔한 사무실 BGM (선택)

이 프로젝트가 "끝났다"고 말하려면 다음을 모두 만족해야 함:

7. ✅ 모바일 브라우저에서 플레이 가능
8. ✅ 도감 또는 통계 화면 1개

이 8개를 만족하면 **그 이상은 안 함**. 손 놓고 다음 프로젝트로.

## 11. 첫 작업 제안 (Claude Code에서 바로 시작)

1. `npm create vite@latest devteam-enhance -- --template vanilla-ts`
2. `npm install phaser`
3. `src/main.ts`에 Phaser 게임 인스턴스 부트스트랩
4. BootScene + MenuScene 만들고 화면에 "Hello 개발팀" 띄우기
5. 캐릭터 도형 1개 그리고 강화 버튼 추가
6. 강화 누르면 콘솔에 결과 찍히는 데까지 (로직만 검증)

여기까지가 Day 1~2 목표. 막히면 Phaser 공식 "Making your first game" 튜토리얼 한 번 따라하면 거의 다 풀림.
